1. ⚡ 기능 소개 한눈에 보기

PikaBuddy는 교수와 학생을 위한 AI 기반 올인원 학습 플랫폼입니다. 다음은 핵심 기능 요약입니다.

📚
강의 관리

강의 개설, 초대 코드/QR로 학생 참가, 캘린더, 강의자료 업로드

📝
AI 과제 생성

주제만 입력하면 AI가 코딩/글쓰기/퀴즈 문제를 자동 생성 (백준/프로그래머스 스타일 포함)

💻
코드 에디터

내장 코드 에디터 + AI 실시간 피드백 + AI 튜터 채팅 + 블록 코딩(Blockly)

✏️
글쓰기 에디터

리치 텍스트 글쓰기 과제 + AI 표현력/논리 분석 피드백

❓
퀴즈

객관식 / 주관식 / 서술형 혼합 퀴즈, 자동 채점, 타이머

📓
노트 에디터

Tiptap 기반 블록 에디터 + 수식 + 드로잉 + Excalidraw + AI 분석 + 실시간 협업

🕸️
지식 그래프

노트 간 연결을 시각화하는 인터랙티브 그래프 (강의별 + 통합 그래프)

💬
메신저

교수 ↔ 학생, 학생 ↔ 학생 메시지, 읽음 확인, 실시간 알림

📊
대시보드

제출 통계, 점수 분포, AI 인사이트, 학생별 상세 분석

👥
팀 협업

팀 생성, 팀 노트 실시간 동시 편집, 팀 과제 투표 제출

🔔
알림 시스템

메시지, 코멘트 통합 알림벨, 클릭 시 해당 위치로 즉시 이동

🎮
게이미피케이션

EXP 시스템, 6단계 티어(씨앗~숲), 과제/챌린지 제출·노트 작성으로 경험치 획득

🛡️
시험 감독

전체화면 강제, 화면 캡쳐, 이탈 감지, 위반 기록, 교수 모니터링

🎨
테마 커스텀

10가지 내장 테마 + 프리셋 + 55가지 이펙트 + CSS/JSON 편집기

🤖
AI 에이전트

학생 AI 튜터 + 교수 AI 조교, 세션 기반 대화, 맥락 인식 답변

2. 🚀 시작하기

2-1. 로그인

PikaBuddy는 두 가지 로그인 방식을 지원합니다.

Google 로그인 — 메인 화면의 Google로 시작하기 버튼을 클릭하면 Google 계정으로 간편 로그인됩니다. 최초 로그인 시 자동으로 계정이 생성됩니다.
관리자 로그인 — 별도 관리자 계정이 있는 경우, 하단의 "관리자 로그인" 링크를 통해 아이디/비밀번호로 접속할 수 있습니다.

2-2. 역할 선택

최초 로그인 시 역할 선택 화면이 나타납니다.

역할설명주요 기능
교수 강의를 개설하고 과제를 관리하는 역할 강의 생성, AI 과제 생성, 대시보드, 학생 노트 리뷰, 채점, 팀 관리
학생 강의에 참가하여 과제를 수행하는 역할 과제 풀기, 노트 작성, AI 피드백, 메신저, 지식 그래프, 팀 협업
개인 강의 없이 혼자 학습하는 모드 자체 챌린지 생성, 노트 작성, 자료 관리, AI 피드백, 개인 대시보드
💡 Tip
역할은 설정(Settings) 페이지에서 언제든 변경할 수 있습니다.

3. 🏠 역할별 홈 화면

3-1. 교수 홈 교수

교수 홈 화면(/professor)에서는 다음을 할 수 있습니다:

  • 내 강의 목록 — 개설한 모든 강의가 카드로 표시됩니다. 강의명, 수강생 수, 초대 코드를 확인할 수 있습니다.
  • 강의 개설 — "강의 만들기" 버튼으로 새 강의를 생성합니다.
  • 캘린더 — 모든 강의의 과제 마감일, 개인 일정, 한국 공휴일이 월별 캘린더에 표시됩니다. 개인 이벤트를 추가/수정/삭제할 수 있습니다.
  • 강의 클릭 — 특정 강의를 클릭하면 커리큘럼(과제 목록) 화면으로 이동합니다.

사이드바 메뉴:

  • 📚 Classroom — 강의 목록 (홈)
  • 💬 Messenger — 강의 밖에서도 바로가기 (최근 대화 코스로 이동)
  • 📝 Notes — 전체 노트 (모든 강의의 노트를 한 곳에서)
  • 📊 Dashboard — 강의 선택 후 대시보드
  • 📝 Curriculum — 강의 선택 후 과제/자료 관리
  • 📓 Student Notes — 학생들의 노트 열람
  • 💬 Messenger — 강의 선택 후 학생과 메시지
  • ⚙ Settings — 계정 설정, 테마, 튜토리얼

3-2. 학생 홈 학생

학생 홈 화면(/student)에서는 다음을 할 수 있습니다:

  • 수강 중인 강의 목록 — 참가한 강의가 카드로 표시됩니다.
  • 강의 참가 — "강의 참가하기" 버튼으로 초대 코드를 입력합니다.
  • 할 일 목록 — 미제출 과제 목록이 우측에 표시됩니다.
  • 캘린더 — 모든 수강 강의의 과제 마감일, 개인 일정, 한국 공휴일이 캘린더에 표시됩니다. 과제를 클릭하면 해당 에디터로 바로 이동합니다.

사이드바 메뉴 (강의 선택 후):

  • 📚 Classroom — 강의 목록 (홈)
  • 💬 Messenger — 강의 밖에서도 바로가기
  • 📝 Notes — 전체 노트
  • 📋 Curriculum — 과제 목록
  • 📝 Notes — 강의별 내 노트
  • 💬 Messenger — 교수/학생과 메시지
  • ⚙ Settings — 계정 설정

3-3. 개인 모드 개인

개인 모드(/personal)는 강의/교수 없이 혼자서 학습하는 모드입니다. 개인 전용 공간이 자동으로 생성됩니다.

  • 대시보드 탭 — 제출 수, 평균 점수, 노트 수, 점수 추이 차트, 챌린지별 성적 차트
  • 챌린지 탭 — 직접 챌린지를 만들고 풀 수 있습니다. AI가 문제를 생성해줍니다.
  • 자료 탭 — PDF 등 학습 자료를 업로드하면 AI 챌린지 생성 시 참고됩니다.
  • 캘린더 — 챌린지 마감일과 개인 일정, 한국 공휴일이 표시됩니다. 이벤트 추가/수정/삭제가 가능합니다.
  • 노트 — 사이드바에서 Notes를 클릭하여 노트를 작성합니다.
  • 지식 그래프 — Graph 메뉴에서 노트 간 연결을 시각화합니다.

4. 🏫 강의 관리

4-1. 강의 개설 교수

교수 홈에서 "강의 만들기" 버튼 클릭
강의명을 입력합니다 (예: "파이썬 프로그래밍 기초")
설명(선택)과 학습 목표(선택, 줄바꿈으로 구분)를 입력합니다. 학습 목표는 AI 노트 분석 시 참고됩니다.
"만들기"를 클릭하면 강의가 생성되고 초대 코드가 자동으로 발급됩니다.

4-2. 수강생 초대 교수

생성된 강의에는 고유한 초대 코드가 부여됩니다.

  • 커리큘럼 페이지 상단에 초대 코드가 표시됩니다.
  • QR 코드 버튼을 클릭하면 QR 코드가 팝업으로 나타나 학생들이 스캔하여 참가할 수 있습니다.
  • 초대 링크(/join/초대코드)를 직접 공유할 수도 있습니다. 링크에 접속하면 강의 정보가 미리보기로 표시됩니다.

4-3. 강의 참가 학생

학생 홈에서 "강의 참가하기" 클릭
교수에게 받은 초대 코드를 입력 (또는 QR 스캔 / 초대 링크 접속)
"참가" 버튼을 누르면 강의에 등록되고 강의 목록에 나타납니다.

4-4. 강의자료 관리

교수 커리큘럼 페이지에서 강의자료를 업로드할 수 있습니다.

  • "자료 업로드" 버튼 클릭 → 파일 선택 (PDF, 이미지, 문서 등)
  • 업로드된 자료는 학생들이 커리큘럼 페이지에서 열람 / 다운로드 가능합니다.
  • 개인 모드에서도 자료 탭에서 동일하게 업로드 가능합니다.

4-5. 캘린더 / 일정

캘린더는 홈 화면과 커리큘럼 페이지 모두에서 사용할 수 있습니다.

  • 홈 화면 캘린더 — 교수, 학생, 개인 모드 모두 홈 화면에 월별 캘린더가 표시됩니다. 모든 강의의 과제 마감일이 통합됩니다.
  • 커리큘럼 캘린더 — 강의 내 커리큘럼 페이지에서 목록 보기와 캘린더 보기를 전환할 수 있습니다.
  • 한국 공휴일 — 음력 포함 한국 공휴일(설, 추석, 부처님오신날 등)이 빨간색으로 표시됩니다.
  • 개인 이벤트 — 날짜를 클릭하여 개인 일정을 추가/수정/삭제할 수 있습니다.
  • 과제 색상 — 과제 유형별로 색상이 구분됩니다 (코딩, 글쓰기, 퀴즈 등). 어느 강의의 과제인지도 표시됩니다.

4-6. 배너 이미지

강의 카드와 상세 페이지에 표시되는 배너 이미지를 설정할 수 있습니다. 12가지 그라디언트 프리셋 중 선택합니다.

🎨 교수 vs 학생 배너
교수가 설정한 배너는 모든 수강생에게 기본 배너로 표시됩니다. 학생은 자신만 볼 수 있는 커스텀 배너를 따로 설정할 수 있으며, 다른 사람에게는 교수가 설정한 기본 배너가 보입니다.

교수: 배너 설정 교수

강의 생성 시 — 새 강의를 만들 때 하단의 그라디언트 프리셋 중 원하는 배너를 선택합니다.
기존 강의 변경 — 강의 카드에 마우스를 올리면 우측 상단에 연필 아이콘이 나타납니다. 클릭하면 배너 선택 팝업이 열립니다.
원하는 프리셋을 선택하고 "저장"을 누르면 해당 강의의 기본 배너가 변경됩니다.

학생: 커스텀 배너 학생

강의 카드에 마우스를 올리면 우측 상단에 연필 아이콘이 나타납니다.
클릭하여 원하는 그라디언트 프리셋을 선택하고 "저장"을 누릅니다.
변경한 배너는 본인에게만 표시됩니다. 다른 수강생이나 교수에게는 보이지 않습니다.
교수의 기본 배너로 되돌리려면 "기본값으로" 버튼을 클릭합니다.

5. 📝 과제 시스템

5-1. AI 과제 생성 교수 개인

PikaBuddy의 핵심 기능 중 하나로, 주제만 입력하면 AI(Gemini)가 문제를 자동 생성합니다.

💡 용어 안내
개인 모드에서는 "과제" 대신 "챌린지"라는 용어를 사용합니다. 기능은 동일하며, 버튼 명칭이 "챌린지 만들기"로 표시됩니다.
커리큘럼 페이지에서 "과제 만들기" (개인 모드: "챌린지 만들기") 클릭
제목과 주제를 입력합니다. 예: 제목 "반복문 연습", 주제 "for문과 while문을 이용한 패턴 출력"
과제/챌린지 유형을 선택합니다: 코딩 / 글쓰기 / 코딩+글쓰기 / 퀴즈
세부 설정을 합니다:
  • 프로그래밍 언어 (Python, JavaScript, Java, C++ 등)
  • 문제 수 (일반 문제 + 백준 스타일 + 프로그래머스 스타일 + 블록 코딩 문제 비율 조절)
  • 난이도 (쉬움 / 보통 / 어려움)
  • 마감일 (선택)
  • AI 정책 & 채점 기준
"만들기"를 누르면 AI가 문제를 생성합니다. 생성 중에는 "생성 중..." 상태가 표시되며, 완료되면 자동으로 목록에 나타납니다.

5-2. 과제 유형

유형설명에디터
코딩 (coding) 프로그래밍 문제를 풀고 코드를 제출합니다. AI가 로직, 품질, 복붙 여부를 분석합니다. 코드 에디터
글쓰기 (writing) 주어진 주제에 대해 글을 작성합니다. AI가 논리 구조, 표현력, 주제 적합도를 분석합니다. 글쓰기 에디터
코딩+글쓰기 (both) 코드 작성과 글쓰기를 모두 포함하는 복합 과제입니다. 코드/글쓰기 에디터
퀴즈 (quiz) 객관식, 주관식, 서술형이 혼합된 퀴즈입니다. 자동 채점됩니다. 퀴즈 에디터

5-3. AI 정책 & 채점 기준 교수

AI 정책 (복붙/외부도구 허용 수준)

정책설명
자유 (free)AI 사용 및 복붙을 허용합니다. 복붙 여부는 참고만 합니다.
보통 (normal)복붙은 감지되며, 과도한 복붙은 감점 요소입니다.
엄격 (strict)AI 사용과 복붙이 제한됩니다. 복붙 발견 시 상당한 감점.
시험 (exam)모든 외부 도구와 복붙이 금지. 전체화면 감독 모드 활성화. 복붙 발견 시 부정행위로 처리.

채점 기준 (채점 엄격도)

기준설명
순한맛 (mild)관대하게 채점. 노력과 시도를 높이 평가. 격려 위주 피드백.
보통맛 (normal)균형 잡힌 채점. 잘한 점은 칭찬, 부족한 점도 명확히 지적.
매운맛 (strict)엄격한 채점. 높은 기준 적용. 80점 이상은 정말 잘한 경우에만.
💡 Tip
교수는 "교수 유의사항(grading_note)"을 추가로 입력할 수 있습니다. 이 메모는 AI 채점 시 반드시 반영됩니다. 예: "들여쓰기를 엄격하게 평가해주세요."

5-4. 과제 관리 교수

생성된 과제에 대해 다양한 관리 작업을 할 수 있습니다:

  • 발행/비발행 — 과제를 "발행"하면 학생에게 공개됩니다. 비발행 상태에서는 학생이 볼 수 없습니다.
  • 문제 수정 — 개별 문제의 내용, 테스트 케이스, 기본 코드를 수정할 수 있습니다.
  • 문제 추가/삭제 — 과제에 문제를 추가하거나 삭제할 수 있습니다.
  • 문제 은행 — 다른 과제에서 문제를 가져올 수 있는 공유 문제 은행입니다.
  • AI 정책 변경 — 과제 생성 후에도 AI 정책, 채점 기준, 글쓰기 프롬프트를 변경할 수 있습니다.
  • 과제 상세 — 과제를 클릭하면 전체 제출 현황, 학생별 점수, 복붙 로그, 코드 스냅샷을 확인할 수 있습니다.

6. 💻 코드 에디터

코딩 과제를 클릭하면 코드 에디터 페이지로 이동합니다.

6-1. 코드 제출 & AI 피드백

좌측에 문제 설명이 표시됩니다. 여러 문제가 있는 경우 상단 탭으로 전환합니다.
우측 코드 에디터에서 코드를 작성합니다. 기본 제공 코드(starter code)가 있으면 미리 입력되어 있습니다.
"제출하기" 버튼을 클릭하면 코드가 제출되고, AI가 실시간 스트리밍으로 피드백을 생성합니다.
AI 피드백에는 다음이 포함됩니다:
  • 📊 추천 점수 (0~100점)
  • 📝 종합 피드백
  • 🔍 로직 분석
  • ✨ 코드 품질 (변수명, 구조, 가독성)
  • 📋 복붙 분석 (외부 복붙 횟수, 비율, 줄 번호)
  • 💡 개선 제안
⚠️ 주의
외부에서 코드를 복사+붙여넣기하면 자동으로 감지됩니다. AI 정책에 따라 감점될 수 있습니다. 스냅샷이 실시간으로 기록됩니다.

6-1b. 코드 실행 & 채점

코드를 제출하기 전에 직접 테스트할 수 있습니다.

  • 실행 (Run) 탭 — 표준 입력(stdin)을 입력하고 코드를 실행하여 출력을 확인합니다. Ctrl+R
  • 채점 (Judge) 탭 — 테스트 케이스를 기반으로 자동 채점합니다. 통과/실패, 실행 시간, 메모리 사용량이 표시됩니다. Ctrl+J
  • 결과에 Passed / Failed / Error / Time Limit 등의 판정(verdict)이 표시됩니다.
  • 히든 테스트 케이스는 입력/출력이 숨겨져 있으며 결과만 표시됩니다.
💡 Tip
코드는 2.5초마다 자동 저장(스냅샷)됩니다. 브라우저를 닫았다가 다시 열어도 마지막 코드가 복원됩니다.

6-2. AI 튜터 채팅 학생 개인

코드 에디터 우측 하단의 채팅 영역에서 AI 튜터와 대화할 수 있습니다.

  • 현재 작성 중인 코드와 문제 맥락을 AI가 이해합니다.
  • "이 코드에서 에러가 발생하는데 왜 그런가요?" 같은 질문을 할 수 있습니다.
  • AI 튜터는 소크라테스식으로 답변합니다 — 직접 정답을 알려주지 않고 질문으로 유도합니다.
  • AI 튜터의 응답도 실시간 스트리밍됩니다.

6-3. 시험 모드 & 감독

AI 정책이 시험(exam)으로 설정된 과제에서는 시험 모드가 활성화됩니다. 코드, 글쓰기, 퀴즈 과제 모두에서 동작합니다.

학생 측 (시험 응시)

과제에 접속하면 시험 안내 화면이 나타납니다. 전체화면 전환과 화면 공유가 필요하다는 안내가 표시됩니다.
"시험 시작" 버튼을 클릭하면 전체화면으로 전환되고, 화면 캡쳐 공유 권한을 요청합니다.
시험 중 상단에 위반 횟수와 "시험 끝내기" 버튼이 표시됩니다.
시험 종료 시 정상 종료(성공 화면) 또는 강제 종료(경고 화면)가 표시됩니다.

이탈 감지 & 위반 기록

  • 전체화면 해제 — ESC 등으로 전체화면을 나가면 위반 기록 + 자동 전체화면 재진입
  • 탭 전환 — 다른 탭으로 전환하면 위반 기록
  • 창 포커스 이탈 — 다른 창으로 이동하면 위반 기록
  • 화면 공유 중단 — 화면 공유를 직접 중단하면 위반 기록
  • 위반이 최대 허용 횟수를 초과하면 시험이 자동 종료됩니다.
⚠️ 주의
시험이 종료되면 재입장이 불가능합니다. 교수가 대시보드에서 별도로 초기화해야 재응시할 수 있습니다.

교수 측 (시험 감독) 교수

  • 시험 설정 — 스크린샷 촬영 간격(초), 최대 위반 허용 횟수, 스크린샷 품질, 전체화면 필수 여부를 설정합니다.
  • 스크린샷 열람 — 학생별 캡쳐된 화면을 시간순으로 확인합니다.
  • 위반 기록 — 학생별 위반 유형(전체화면 해제, 탭 전환, 포커스 이탈 등)과 시간을 확인합니다.
  • 시험 요약 — 학생별 스크린샷 수, 위반 횟수, 시험 상태(완료/미응시)를 한눈에 확인합니다.
  • 시험 초기화 — 특정 학생의 시험을 초기화하여 재응시를 허용합니다 (감사 로그 기록).

6-4. 블록 코딩 (Blockly)

과제에 블록 코딩 문제가 포함된 경우, 비주얼 블록 프로그래밍 에디터가 표시됩니다.

  • Blockly 기반의 드래그 앤 드롭 인터페이스
  • 블록을 조합하여 로직을 구성하면 코드가 자동 생성됩니다.
  • 프로그래밍 초보자에게 적합한 방식입니다.

7. ✏️ 글쓰기 에디터

글쓰기 과제를 클릭하면 리치 텍스트 에디터가 열립니다.

  • 서식 도구 (굵게, 기울임, 밑줄, 리스트 등)를 사용할 수 있습니다.
  • 글자 수가 실시간으로 표시됩니다.
  • 제출하면 AI가 다음을 분석합니다:
    • 📖 논리 구조 (서론-본론-결론)
    • ✍️ 표현력 (어휘 다양성, 문법)
    • 🎯 주제 적합도
    • 📋 복붙 분석
    • 💡 개선 제안
  • 시험 모드(exam) 과제에서는 시험 감독 기능이 동일하게 적용됩니다.

8. ❓ 퀴즈 시스템

퀴즈 과제를 클릭하면 퀴즈 에디터가 열립니다.

  • 객관식 (MC) — 보기 중 정답을 선택합니다.
  • 주관식 (SA) — 짧은 답을 직접 입력합니다.
  • 서술형 (Essay) — 긴 텍스트로 서술합니다.

문제 수 비율은 과제 생성 시 교수가 설정합니다 (기본: 객관식 5 / 주관식 3 / 서술형 2).

  • 제출 시 객관식과 주관식은 자동 채점됩니다.
  • 서술형은 AI가 분석하여 피드백을 제공합니다.
  • 타이머가 설정된 경우 시간 제한이 적용됩니다.
  • 시험 모드(exam) 과제에서는 시험 감독 기능이 동일하게 적용됩니다.

9. 📓 노트 에디터

PikaBuddy의 가장 풍부한 기능을 갖춘 에디터입니다. Tiptap 기반 블록 에디터로, 다양한 콘텐츠를 작성할 수 있습니다.

9-1. 블록 & 서식

표 편집

표 안에 커서를 놓으면 상단 툴바에 "표" 드롭다운이 나타납니다:

  • 열 추가 / 열 삭제
  • 행 추가 / 행 삭제
  • 셀 병합/분리
  • 표 삭제

사이드바 탭

노트 에디터 우측에는 4개의 사이드바 탭이 있습니다:

  • AI — AI 분석 결과 + AI 튜터 채팅
  • 자료 — 강의 자료(PDF, PPT 등) 임베드 보기. Office 365 뷰어로 렌더링됩니다.
  • 노트 맵 — 현재 노트의 부모/자식 계층 구조를 트리로 표시
  • 코멘트 — 이 노트에 달린 코멘트 스레드 목록 (리뷰 모드에서 활성화)

텍스트 서식:

  • Ctrl+B 굵게 / Ctrl+I 기울임 / Ctrl+U 밑줄
  • Ctrl+Shift+X 취소선
  • Ctrl+E 인라인 코드
  • Ctrl+K 링크 삽입
  • 텍스트 선택 시 버블 메뉴가 나타나 서식을 적용할 수 있습니다.
  • 형광펜, 글자 색상, 위첨자, 아래첨자, 텍스트 정렬도 지원됩니다.

9-2. AI 분석 & AI 다듬기

AI 갭 분석 (Ctrl+Enter)

노트를 저장한 후 Ctrl+Enter를 누르거나 "AI 분석" 버튼을 클릭하면 AI가 노트를 분석합니다.

  • 📊 이해도 점수 (0~100점) — 강의 목표 대비 이해도
  • 📝 종합 평가
  • ✅ 잘 이해한 부분
  • ⚠️ 보완이 필요한 부분
  • 💡 학습 추천
  • 🏷️ 카테고리 — AI가 자동으로 분류 (예: variables, loops, functions 등)

분석 결과는 사이드바의 "AI 분석" 탭에 표시되며, 실시간 스트리밍으로 결과가 나타납니다.

💡 Tip
노트에 AI 다듬기로 처리된 구간이 있으면, AI 분석에서 자동으로 제외됩니다. 학생이 직접 작성한 부분만 평가됩니다.

AI 인라인 코멘트

AI 분석 완료 후, 노트 본문에 인라인 코멘트가 표시됩니다.

  • 초록색 밑줄 — 정확하게 이해한 내용
  • 빨간색 밑줄 — 부정확하거나 보완이 필요한 내용
  • 밑줄 위에 마우스를 올리면 AI의 구체적인 코멘트가 표시됩니다.

AI 다듬기

노트 본문에서 텍스트를 선택하면 버블 메뉴에 "AI 다듬기" 버튼이 나타납니다.

  • 선택한 텍스트를 AI가 문법, 표현, 논리적으로 다듬어줍니다.
  • 다듬어진 텍스트는 별도 블록으로 표시되며, AI 분석에서 제외됩니다.
  • 원본 텍스트와 다듬어진 텍스트를 비교할 수 있습니다.

9-3. 드로잉 & 수식

Excalidraw 드로잉

/ → 그리기를 선택하면 Excalidraw 기반 화이트보드가 삽입됩니다.

  • 도형, 화살표, 텍스트를 이용한 다이어그램 작성
  • 플로우차트, 개념도, 아키텍처 다이어그램 등에 유용합니다.
  • 자유 드로잉, 펜, 지우개, 색상 선택 지원

수식 (LaTeX)

  • 인라인 수식: / → 수식. 텍스트 중간에 수식 삽입 (예: E = mc^2)
  • 블록 수식: / → 수식 블록. 별도 줄에 큰 수식 표시
  • LaTeX 문법을 사용하며, KaTeX로 실시간 렌더링됩니다.

9-4. 노트 링크 & 그래프

노트 링크

노트 에디터에서 노트 간 상호 참조를 만드는 두 가지 방법이 있습니다:

  • [[를 입력하면 노트 링크 검색 팝업이 나타납니다.
  • / → 노트 링크를 선택해도 동일한 검색 팝업이 열립니다.
  • 같은 강의 내 다른 노트를 검색하여 링크를 삽입합니다.
  • 링크된 노트를 클릭하면 해당 노트로 이동합니다.
  • 이 연결 관계가 지식 그래프에 반영됩니다.

지식 그래프

사이드바의 Graph 메뉴를 클릭하면 노트 간 연결을 인터랙티브 포스 그래프로 시각화합니다.

  • 각 노트가 노드로 표시되고, 노트 링크가 엣지로 표시됩니다.
  • 노드 크기는 이해도 점수에 비례합니다.
  • 노드 색상은 점수별로 다릅니다: 🔴 40점 미만 / 🟡 40~60 / 🔵 60~80 / 🟢 80 이상 / 📝 분석 전
  • 엣지 유형: 부모-자식 / 사용자 링크 / AI 유사도 (각각 토글 가능)
  • 필터: 점수 범위, 태그, 날짜 범위로 노드를 필터링
  • 링킹 모드: 노드를 클릭하여 새 연결을 만들 수 있습니다.
  • 노드를 클릭하면 해당 노트로 이동합니다.
  • 사이드 패널에서 AI 학습 경로와 주간 리포트를 확인할 수 있습니다.

9-5. 스냅샷 & 버전 비교

노트는 저장할 때마다 스냅샷(버전)이 자동으로 기록됩니다.

  • 노트 에디터 상단의 스냅샷 아이콘을 클릭하면 버전 히스토리가 열립니다.
  • 과거 스냅샷을 선택하면 현재 내용과의 Diff 비교를 볼 수 있습니다.
  • 추가된 블록, 삭제된 블록, 변경된 블록이 색상으로 구분됩니다.
  • 특정 시점의 노트 상태를 확인할 수 있습니다.

9-6. 실시간 협업

팀 노트에서는 팀원들이 동시에 같은 노트를 편집할 수 있습니다.

  • 다른 팀원이 편집 중이면 상단에 "OOO 편집 중" 인디케이터가 표시됩니다.
  • 변경 사항이 실시간으로 동기화됩니다 (Supabase Realtime 기반).
  • 동기화 중... 표시가 나타나며, 충돌 없이 동시 편집이 가능합니다.

11. 👥 팀 & 협업

PikaBuddy는 팀 기반 학습을 지원합니다. 교수가 팀을 구성하면, 팀원들은 노트를 함께 편집하고 과제를 합의하여 제출할 수 있습니다.

11-1. 팀 관리 교수

강의 내 사이드바에서 팀 관리 페이지에 접근합니다.

  • 팀 생성 — 팀 이름을 입력하고 수강생 중에서 팀원을 선택합니다.
  • 팀 수정 — 팀 이름 변경, 팀원 추가/제거가 가능합니다.
  • 팀 삭제 — 팀을 삭제하면 팀 멤버 정보도 함께 삭제됩니다.
  • 팀 목록 — 모든 팀과 구성원을 한눈에 확인합니다.

11-2. 팀 투표 제출 학생

팀 과제에서는 과제를 제출하기 전에 팀원 전원의 동의가 필요합니다.

팀원 중 한 명이 "제출 투표 시작" 버튼을 클릭합니다.
투표가 시작되면 10분 카운트다운이 표시됩니다. 투표를 시작한 사람은 자동으로 찬성됩니다.
다른 팀원들이 찬성(Approve) 또는 반대(Reject)를 투표합니다.
전원 찬성 시 자동으로 과제가 제출되고, 모든 팀원에게 제출 기록이 생성됩니다. 시간 초과 또는 반대가 있으면 투표가 거부됩니다.
💡 Tip
1인 팀은 투표 없이 바로 제출됩니다. 투표 상태는 4초마다 자동 갱신됩니다.

12. 💬 메신저

교수 학생 같은 강의 내에서 메시지를 주고받을 수 있습니다. (5초마다 자동 새로고침)

  • 좌측: 대화 목록 — 상대방 이름, 마지막 메시지, 안 읽은 메시지 수 표시
  • 우측: 채팅 화면 — 메시지를 입력하고 전송
  • 교수는 해당 강의의 모든 수강생에게 메시지를 보낼 수 있습니다.
  • 학생은 해당 강의의 교수와 같은 강의 수강생에게 메시지를 보낼 수 있습니다.
  • 사이드바에 안 읽은 메시지 뱃지가 표시됩니다.
  • 메시지를 열면 자동으로 읽음 처리됩니다.
💡 Tip
강의를 선택하지 않은 상태에서도 사이드바의 Messenger 링크를 통해 가장 최근 대화가 있는 강의로 바로 이동할 수 있습니다.

13. 🔔 알림 시스템

전체 화면 우측 상단의 알림 벨 아이콘을 클릭하면 통합 알림 패널이 열립니다.

  • 안 읽은 메시지와 미해결 코멘트가 통합되어 표시됩니다.
  • 각 알림에는 보낸 사람, 미리보기, 시간(방금/N분 전/N시간 전/N일 전)이 표시됩니다.
  • 알림을 클릭하면 해당 위치로 즉시 이동합니다:
    • 메시지 알림 → 해당 메신저 대화 화면
    • 코멘트 알림 → 해당 노트의 코멘트 블록 위치
  • "모두 읽음" 버튼으로 모든 알림을 한 번에 처리합니다.
  • 알림은 30초마다 자동 갱신됩니다.
  • 알림 벨에 빨간 뱃지가 표시되어 미확인 알림 수를 알려줍니다.

14. 📝 전체 노트 & 통합 그래프

교수 학생 사이드바의 Notes 메뉴(강의 밖)를 클릭하면 모든 강의의 노트를 한 곳에서 볼 수 있습니다.

전체 노트 (/all-notes)

  • 모든 강의의 노트가 통합 목록으로 표시됩니다.
  • 검색 — 노트 제목으로 검색
  • 강의 필터 — 특정 강의의 노트만 필터링
  • 정렬 — 수정일, 생성일, 이해도 점수순 정렬
  • 보기 모드 — 리스트 보기와 그리드 보기 전환
  • 각 노트에 강의명, 이해도 점수, 카테고리, 미리보기가 표시됩니다.

통합 지식 그래프 (/all-notes/graph)

  • 모든 강의의 노트를 하나의 그래프에서 시각화합니다.
  • 강의별 노드를 색상으로 구분합니다.
  • AI 유사도 기반 연결을 포함하여 강의 간 지식 연결도 표시됩니다.
  • 통합 학습 경로와 주간 리포트도 제공됩니다.

15. 📊 대시보드 & 분석

교수 대시보드 교수

강의를 선택한 후 사이드바의 Dashboard를 클릭합니다.

  • 수강생 수, 과제 수, 총 제출 수, 위기 학생 수 등 기본 통계
  • 과제별 평균 점수 차트 (바 차트)
  • 학생 목록 — 각 학생의 제출 현황, 평균 점수 확인
  • 학생 클릭 → 학생 상세 페이지 (개별 제출물, 복붙 로그, 코드 스냅샷, AI 분석 결과)
  • AI 인사이트 — AI가 클래스 전체를 분석하여 다음을 제공:
    • 학습 인사이트 (전체적인 경향)
    • 공통 어려움 (학생들이 공통적으로 어려워하는 부분)
    • 수업 추천 (다음 수업에서 다루면 좋을 내용)

교수 — 점수 확정 교수

학생 상세 페이지에서 각 제출물의 AI 추천 점수를 확인한 뒤, 교수가 최종 점수를 확정할 수 있습니다.

  • AI 추천 점수는 참고용이며, 교수가 직접 점수를 수정하고 확정합니다.
  • 확정된 점수가 학생에게 최종 성적으로 반영됩니다.

개인 대시보드 개인

개인 모드 홈의 대시보드 탭에서 확인합니다.

  • 총 챌린지 수, 총 제출 수, 노트 수, 평균 점수, 최고 점수, 문제 해결률
  • 점수 추이 라인 차트
  • 챌린지별 성적 바 차트
  • 최근 제출물 목록

16. 📓 학생 노트 리뷰 교수

교수는 사이드바의 Student Notes 메뉴에서 수강생들의 노트를 열람할 수 있습니다.

  • 학생별로 노트 목록이 표시됩니다. 각 노트의 코멘트 수도 확인 가능합니다.
  • 노트를 클릭하면 리뷰 모드로 열립니다 (읽기 전용).
  • 리뷰 모드에서도 코멘트를 남길 수 있습니다.
  • 학생의 AI 분석 결과(이해도 점수 등)도 함께 확인할 수 있습니다.

17. 💬 코멘트 시스템

교수 학생 노트에 코멘트를 남길 수 있습니다.

노트 전체 코멘트

  • 노트 에디터 사이드바의 "코멘트" 탭에서 노트 전체에 대한 코멘트를 작성합니다.
  • 교수와 학생 모두 코멘트를 남길 수 있습니다.
  • 대댓글(답글) 기능이 지원됩니다.

블록별 코멘트

  • 특정 블록(문단)에 대해 개별 코멘트를 남길 수 있습니다.
  • 블록 옆 거터(gutter)에 코멘트 수가 표시됩니다.
  • 알림에서 코멘트를 클릭하면 해당 블록 위치로 자동 스크롤됩니다.

코멘트 관리

  • 수정: 본인이 작성한 코멘트만 수정 가능
  • 삭제: 본인 코멘트 또는 교수가 삭제 가능
  • 해결 처리: 교수 또는 노트 작성자가 코멘트를 "해결됨"으로 표시 가능

18. 🤖 AI 에이전트

PikaBuddy는 역할별 전용 AI 에이전트를 제공합니다. 세션 기반으로 대화 맥락이 유지됩니다.

학생 AI 에이전트 학생 개인

  • 소크라테스식 튜터 — 직접 정답을 알려주지 않고, 질문과 힌트로 이해를 유도합니다.
  • 현재 과제, 코드, 문제 맥락을 이해하고 맞춤 답변을 제공합니다.
  • 개념 설명(재귀, 알고리즘 등)은 정답 없이도 설명해줍니다.
  • 대화 히스토리가 세션 동안 유지됩니다.

교수 AI 에이전트 교수

  • AI 조교 — 교수의 수업 준비와 학생 관리를 돕습니다.
  • 과제 자료 작성, 성적 분석, 피드백 제안, 채점 기준 설계를 지원합니다.
  • 대화 히스토리가 세션 동안 유지됩니다.

세션 관리

  • 세션 정보 확인 및 대화 기록 열람이 가능합니다.
  • 세션을 초기화하면 대화 기록이 삭제되고 새로운 대화가 시작됩니다.

19. 🎮 게이미피케이션

학생 개인 PikaBuddy는 학습 동기 부여를 위한 EXP/티어 시스템을 갖추고 있습니다.

경험치(EXP) 획득

  • 과제/챌린지 제출 시 EXP 획득
  • 노트 작성 및 AI 분석 시 EXP 획득
  • 활동량에 따라 누적됩니다.

티어 시스템

티어아이콘의미
씨앗 (Seed)🌱학습을 시작하는 단계
새싹 (Sprout)🌿기초를 다지는 단계
나무 (Tree)🌳실력이 자라는 단계
꽃 (Bloom)🌸실력이 개화하는 단계
열매 (Fruit)🍎성과를 맺는 단계
숲 (Forest)🌲최고 경지에 도달한 단계

사이드바 하단에 현재 티어와 EXP 진행률이 표시됩니다. 각 티어에는 I~IV 세부 등급이 있어 총 24단계입니다.

💡 Tip
특정 테마에서는 레벨업 시 컨페티(confetti) 이펙트가 발동됩니다!

20. 👤 프로필

전체 사이드바의 프로필 영역을 클릭하거나 /profile/사용자ID로 접근합니다.

프로필 페이지 구성

  • 배너 — 프로필 배너 이미지 (설정에서 업로드) 또는 프로필 색상 기반 그라데이션 배경
  • 아바타 — 원형 프로필 이미지 (설정에서 업로드 & 크롭)
  • 기본 정보 — 이름, 역할 뱃지(교수/학생/개인), 학교, 학과
  • 자기소개 — 설정에서 입력한 소개글
  • 소셜 링크 — GitHub, 블로그 URL (클릭 가능)
  • 가입일 — 계정 생성일
💡 Tip
다른 사용자의 프로필도 볼 수 있습니다. 대시보드 학생 목록이나 메신저에서 이름을 클릭하면 해당 학생의 프로필로 이동합니다.

21. 🎨 테마 & 이펙트 커스텀

전체 설정(Settings)의 외관 섹션에서 앱의 전체 디자인을 바꿀 수 있습니다.

내장 테마

10가지 내장 테마가 제공됩니다. 클릭 한 번으로 적용됩니다.

  • 기본(Default), 다크, 코딩, 사이버펑크, 파스텔, 네이처, 오션, 선셋, 라벤더 등
  • 각 테마 카드의 ✏ 아이콘을 클릭하면 해당 테마를 기반으로 커스텀 테마를 만들 수 있습니다.

프리셋 커스텀 테마

네온 갤럭시, 오션 딥, 체리 블라썸 등 이펙트가 포함된 프리셋 테마를 한 번에 적용할 수 있습니다.

커스텀 테마 만들기

"만들기" 카드를 클릭하면 테마 에디터가 열립니다:

  • GUI 에디터 탭 — 47개 CSS 변수를 색상 피커/슬라이더로 시각적으로 편집. 기본 테마에서 시작 가능.
  • 이펙트 탭 — 55개 이펙트 중 원하는 것을 활성화하고 파라미터를 조절
    • 배경: 파티클, 별빛, 오로라, 매트릭스 레인, 버블, 벚꽃, 낙엽, 번개, 안개
    • 패턴: 도트 그리드, 노이즈, 기하학 패턴, 마우스 그라데이션
    • UI: 글로우, 글래스모피즘, 그라데이션 테두리, 카드 틸트, 소프트 섀도우 등
    • 텍스트: 글리치, 레인보우, 네온, 물결, 타자기 효과 등
    • 커서: 마우스 트레일, 커서 글로우, 커스텀 커서, 클릭 폭발 등
    • 애니메이션: 페이드인 스크롤, 페이지 전환, 스켈레톤 쉬머 등
  • JSON 편집 탭 — 테마 JSON을 직접 편집
  • CSS 주입 탭 — 커스텀 CSS를 직접 작성하여 앱 스타일 오버라이드
💡 Tip
테마를 .pikabuddy-theme.json 파일로 내보내기/가져오기할 수 있습니다. 친구에게 공유하거나 백업하세요.
⚠️ 성능 참고
이펙트를 많이 동시에 활성화하면 성능이 저하될 수 있습니다. 자동 성능 모니터가 FPS를 감지하여 파티클 수를 조절하지만, 저사양 기기에서는 이펙트를 적절히 사용하세요.
