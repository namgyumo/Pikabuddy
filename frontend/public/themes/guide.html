1. 테마 JSON 구조 (5계층)

PikaBuddy 테마는 .pikabuddy-theme.json 파일 하나에 5개 계층이 모두 담깁니다.

{
  "name": "My Theme",         // 테마 이름 (필수)
  "version": 1,                  // 항상 1 (필수)

  // [계층 1] GUI 변수 — 47개 CSS 변수
  "variables": {
    "--primary": "#A855F7",
    "--surface": "#0B0E1A",
    ...
  },

  // [계층 2] Custom CSS — 세밀한 스타일 오버라이드
  "customCSS": ".card:hover { box-shadow: 0 0 20px rgba(168,85,247,0.3); }",

  // [계층 3] 애니메이션 프리셋
  "animation": "particles",

  // [계층 4] 상시 이펙트 — layer로 겹침 순서 제어
  "effects": {
    "starfield": { "enabled": true, "params": { "speed": 0.75 }, "layer": 0 },
    "aurora":    { "enabled": true, "params": { ... }, "layer": 1 },
    "glow":      { "enabled": true, "params": { ... }, "layer": 2 }
  },

  // [계층 5] 이벤트 트리거 — 정답/오답/레벨업 시 발동
  "triggers": {
    "onCorrect": ["confetti", "correctBounce"],
    "onWrong":    ["wrongShake"],
    "onLevelUp": ["levelUpCelebration"]
  }
}
필수 필드: name, version, variables (최소 1개 이상)
선택 필드: customCSS, animation, effects, triggers

3. Custom CSS 작성법

customCSS 필드에 CSS를 넣으면 페이지에 직접 주입됩니다.

보안 제한: @import, expression(), javascript:, <script> 태그는 자동 제거됩니다.

자주 쓰는 셀렉터

셀렉터대상
.card, .assignment-card, .note-card카드 컴포넌트
.card:hover카드 호버
h1, h2, h3, .page-title제목
.btn-primary, button[class*="primary"]주 버튼
.sidebar, [class*="sidebar"]사이드바
::-webkit-scrollbar-thumb스크롤바
::selection텍스트 선택 영역

인기 CSS 패턴

그라디언트 텍스트

h1, h2 {
  background: linear-gradient(135deg, #EC4899, #A855F7);
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
  background-clip: text;
}

유리 효과 카드

.card {
  background: rgba(255, 255, 255, 0.1) !important;
  backdrop-filter: blur(16px) saturate(1.4);
  border: 1px solid rgba(255, 255, 255, 0.1) !important;
}

네온 글로우 버튼

.btn-primary {
  box-shadow: 0 0 15px rgba(168, 85, 247, 0.4),
              inset 0 1px 0 rgba(255, 255, 255, 0.1);
}

그라디언트 스크롤바

::-webkit-scrollbar-thumb {
  background: linear-gradient(180deg, #A855F7, #6366F1) !important;
  border-radius: 10px;
}

4. 애니메이션 프리셋

animation 필드에 다음 ID 중 하나를 지정합니다.

ID이름설명
"none"없음애니메이션 비활성화
"particles"떠다니는 입자부유하는 파티클
"rain"비빗줄기 효과
"sparkle"반짝임별 반짝임
"snow"눈눈 내리는 효과
"gradient"그라디언트 흐름색상이 흐르는 배경

6. 이미지/영상 에셋 활용법

커서 이미지, 배경 미디어(이미지/GIF/동영상), 마스코트 스프라이트를 활용하면 완전히 독창적인 테마를 만들 수 있습니다.

6.1 커서 이미지 (customCursorImage)

마우스 커서를 나만의 이미지로 교체합니다. 3가지 상태별로 다른 커서를 지정할 수 있습니다.

파라미터적용 대상권장 크기
default일반 영역 (배경, 본문)32x32px
pointer클릭 가능 요소 (버튼, 링크, 카드)32x32px
text텍스트 입력 영역 (인풋, 에디터)32x32px
주의: 커서 이미지는 128x128px 이하여야 하며, 32x32px PNG가 가장 안정적입니다. 투명 배경 PNG를 사용하세요. 
"customCursorImage": {
  "enabled": true,
  "params": {
    "default": "https://example.com/cursor-normal.png",
    "pointer": "https://example.com/cursor-pointer.png",
    "text": "https://example.com/cursor-text.png"
  }
}

6.2 배경 미디어 (backgroundImage) — 이미지 / GIF / 동영상

배경에 이미지, GIF 애니메이션, 동영상(mp4/webm)을 오버레이합니다. 미디어 타입은 URL에서 자동 감지됩니다.

파라미터설명옵션
url미디어 URL 또는 data URIPNG, JPG, SVG, WebP, GIF, MP4, WebM
opacity불투명도0.02 ~ 1 (기본 0.15)
blendMode혼합 모드normal, multiply, screen, overlay, soft-light, luminosity
size이미지/영상 크기cover(채우기), contain(맞추기), auto(원본), 200px/400px(타일)
position위치 정렬center, top, bottom, left, right

미디어 타입별 동작

타입감지 기준렌더링 방식
이미지 (PNG, JPG 등)기본값background-image — 정적 오버레이
GIF확장자 .gif 또는 data:image/gifbackground-image — 브라우저가 자동 재생
동영상 (MP4, WebM)확장자 .mp4 / .webm 또는 data:video/<video> 요소 — 자동 반복 재생, 음소거
// 정적 이미지
"backgroundImage": {
  "enabled": true,
  "params": {
    "url": "https://example.com/bg-pattern.png",
    "opacity": 0.1,
    "blendMode": "soft-light"
  }
}

// GIF 애니메이션 배경
"backgroundImage": {
  "enabled": true,
  "params": {
    "url": "https://example.com/rain-loop.gif",
    "opacity": 0.2,
    "size": "cover"
  }
}

// 동영상 배경 (자동 반복, 음소거)
"backgroundImage": {
  "enabled": true,
  "params": {
    "url": "https://example.com/ambient-loop.mp4",
    "opacity": 0.15,
    "blendMode": "screen",
    "size": "cover"
  }
}
활용 팁:
  • 타일 모드(200px/400px) + 패턴 이미지 = 반복 배경
  • opacity 0.05~0.15 + multiply 블렌드 = 은은한 텍스처
  • GIF: 루프 애니메이션 배경 (비/눈/파도 등) — 파일 크기 주의
  • 동영상: 고화질 앰비언트 루프 — 외부 URL 권장 (data URI는 용량이 큼)

마스코트 기능:
  • CSS steps() 애니메이션으로 구현 — GPU 부하 거의 없음
  • image-rendering: pixelated — 픽셀아트 캐릭터도 선명하게 표시
  • 3가지 시트 배치 — 가로(horizontal), 세로(vertical), 격자(grid) 모두 지원
  • 마우스/터치 드래그 지원 — 원하는 위치로 이동 가능 (드래그 중 자동 일시정지, 놓으면 재개)
  • 30개 행동 함수 — walkTo, bounce, say, emote, playSprite, spawn/kill 등
  • JSON 스크립트 — 시작 동작, 루프, 이벤트 리액션을 프로그래밍
  • 멀티 스프라이트 — 상태별로 다른 스프라이트 시트 전환 (idle/walk/happy 등)
  • 콘솔에서 __pikabuddyMascot.dance() 같은 직접 호출도 가능

스프라이트 시트 구하기

  • 직접 그리기: Aseprite, Piskel(무료 웹), LibreSprite로 픽셀아트 제작
  • AI 생성: "pixel art sprite sheet, 4 frames walk cycle, transparent background" 프롬프트
  • 무료 에셋: OpenGameArt, itch.io의 무료 스프라이트 검색
  • 크기: 32x32 ~ 128x128px 권장, scale 파라미터로 확대/축소 가능
  • 멀티 시트: idle/walk/happy 등 여러 동작별 시트를 준비 → sprites에 등록
스프라이트 시트 vs 정지 이미지: 정지 이미지 1장만 있어도 OK! frameCount: 1로 설정하면 프레임 애니메이션 없이 위치 이동/대사/이모지 등 행동 함수만 사용됩니다.

7.2 JSON 스크립트 구조

script 파라미터에 JSON 문자열을 넣어 마스코트 행동 시나리오를 구성합니다.

{
  // 여러 스프라이트 시트 등록 (선택)
  "sprites": {
    "idle":  { "url": "idle.png",  "frameCount": 4, "frameWidth": 64, "frameHeight": 64, "fps": 6 },
    "walk":  { "url": "walk.png",  "frameCount": 6, "frameWidth": 64, "frameHeight": 64, "fps": 10, "layout": "vertical" },
    "happy": { "url": "happy.png", "frameCount": 12, "frameWidth": 64, "frameHeight": 64, "fps": 8, "layout": "grid", "cols": 4 }
  },

  // 시작 시 1회 실행 (순차)
  "onStart": [
    { "action": "spawn", "params": { "name": "idle" } },
    { "action": "say", "params": { "text": "안녕! 👋", "duration": 2000 } },
    { "action": "bounce", "params": { "height": 15, "count": 2 } }
  ],

  // 반복 루프 (무한 반복)
  "loop": [
    { "action": "playSprite", "params": { "name": "idle" } },
    { "action": "idle", "params": { "duration": 8000 } },
    { "action": "playSprite", "params": { "name": "walk" } },
    { "action": "wander", "params": { "steps": 2 } }
  ],

  // 이벤트 리액션
  "reactions": {
    "onClick": [
      { "action": "playSprite", "params": { "name": "happy" } },
      { "action": "bounce", "params": { "height": 20 } },
      { "action": "emote", "params": { "emoji": "😊" } }
    ],
    "onCorrect": [
      { "action": "playSprite", "params": { "name": "happy" } },
      { "action": "jump", "params": { "height": 40 } },
      { "action": "dance", "params": { "duration": 1500 } }
    ],
    "onWrong": [
      { "action": "shake" },
      { "action": "emote", "params": { "emoji": "😢" } }
    ],
    "onLevelUp": [
      { "action": "spin", "params": { "turns": 2 } },
      { "action": "say", "params": { "text": "레벨업! 🎊" } }
    ],
    "onHover": [{ "action": "nod" }],
    "onIdle": [{ "action": "sleep", "params": { "duration": 5000 } }]
  }
}

7.3 실행 흐름

마스코트 활성화
  ├── sprites 등록 (idle, walk, happy 등)
  ├── onStart 실행 (순차) → spawn → 말풍선 → 바운스
  ├── loop 반복 실행 → idle 대기 → walk 스프라이트 → 배회 → ...
  └── reactions 대기
       ├── 클릭 → happy 스프라이트 + bounce + emote
       ├── 정답 → happy + jump + dance
       ├── 오답 → shake + emote 😢
       ├── 레벨업 → spin + say "레벨업!"
       ├── 호버 → nod
       └── 30초 무입력 → sleep
드래그 안전장치: 마스코트를 드래그하면 현재 애니메이션이 자동 일시정지되고, 놓으면 루프가 재개됩니다. 드래그 vs 클릭은 3px 이동 기준으로 자동 구분됩니다.

7.4 spawn/kill로 동적 등장/퇴장

spawn과 kill을 조합하면 마스코트가 이벤트에 맞춰 등장/퇴장하는 연출이 가능합니다.

// 정답 시: 다른 캐릭터로 변신 → 축하 → 원래로 복귀
"onCorrect": [
  { "action": "kill" },
  { "action": "spawn", "params": { "name": "happy" } },
  { "action": "dance", "params": { "duration": 2000 } },
  { "action": "kill" },
  { "action": "spawn", "params": { "name": "idle" } }
]

10. 성능 자동 조절

PikaBuddy 테마 엔진은 자동 성능 관리 기능이 내장되어 있습니다.

통합 애니메이션 루프

Canvas 이펙트(particles, starfield, aurora, matrixRain, bubbles)는 하나의 requestAnimationFrame 루프에서 함께 처리됩니다. 이펙트 5개를 켜도 RAF는 1개만 돌아갑니다.

FPS 자동 감소

실시간 FPS를 측정하여 30fps 이하로 떨어지면 파티클 밀도를 자동 축소합니다 (최소 25%). 50fps 이상으로 회복되면 점진적으로 복원합니다.

탭 비활성화 일시정지

브라우저 탭이 비활성화되면 모든 이펙트 렌더링이 자동 일시정지되고, 다시 활성화되면 재개됩니다.

CSS 이펙트 우선

glow, glassMorphism, gradientBorder, cherryBlossom, fogMist 등은 CSS 애니메이션으로 처리되어 GPU 가속을 활용합니다.

이펙트를 많이 켤수록 파티클이 자동으로 줄어듭니다. 저사양 기기에서도 걱정 없이 사용하세요.

13. 디자인 팁

색상 대비 확인

--on-surface(텍스트)와 --surface(배경) 사이에 충분한 대비(4.5:1 이상)를 유지하세요. 다크 테마에서 특히 중요합니다.

이펙트는 3~5개가 적절

라이트/다크 테마 구분
